## 二、C++ 与 C 的关系

### 2.1 发展背景

- C++ 源于 C 语言
- C++ 保留了 C 的高效性和简洁性
- C++ 在 C 的基础上扩展了面向对象和更灵活的编程机制

### 2.2 本质理解

- C++ 兼容大部分 C 代码
- C++ 既支持面向过程，也支持面向对象
- C++ 不是“纯面向对象语言”

### 2.3 相对 C 的主要扩展

- `new` / `delete`：动态内存管理
- 引用 `&`：更方便的参数传递
- 函数重载：同名函数可按参数区分
- 默认参数：提高调用灵活性
- 内联函数 `inline`：减少函数调用开销
- 类与对象：支持封装、继承、多态
- 异常处理、模板、标准库（STL）
- 流式输入输出：`cin` / `cout`

### 2.4 考试常见表述

- C 程序强调“数据 + 算法”
- C++ 面向对象程序强调“对象 + 消息”

## 三、C++ 词法基础

### 3.1 字符集

C++ 程序由以下基本元素组成：

- 大小写英文字母
- 数字字符 `0~9`
- 各类运算符和分隔符
- 空格、换行、制表符等空白符

### 3.2 词法记号

程序由以下词法单元构成：

- 关键字
- 标识符
- 常量（文字常量）
- 运算符
- 分隔符
- 空白符

### 3.3 关键字

- 关键字是 C++ 预定义保留字
- 关键字不能作为变量名、函数名、类型名使用

常见关键字示例：

- `int` `char` `double` `bool`
- `if` `else` `switch` `for` `while`
- `class` `struct` `public` `private`
- `new` `delete`
- `return` `void`

### 3.4 标识符

标识符命名规则：

- 只能由字母、数字、下划线组成
- 只能以字母或下划线开头
- 区分大小写
- 不能与关键字重名

可表示：

- 变量名
- 常量名
- 函数名
- 数组名
- 类型名

### 3.5 常量

#### 数值常量

- 整型：如 `100`、`200U`、`10000L`
- 十六进制：如 `0x100`
- 浮点型：如 `3.14`、`1.25e+10`

#### 字符常量

- 用单引号括起来，如 `'a'`
- 本质上是整数（字符编码值）

#### 字符串常量

- 用双引号括起来，如 `"China"`、`"C++ Program"`
- 字符串结尾默认带空字符 `\0`

### 3.6 注释

C++ 支持两种注释：

- 多行注释：`/* ... */`
- 单行注释：`// ...`

作用：

- 便于阅读、理解、维护
- 注释内容不参与编译

## 四、基本数据类型与用户自定义类型

### 4.1 内置基本类型

常见内置类型：

- `char`
- `short`
- `int`
- `long`
- `float`
- `double`
- `bool`
- 指针类型（如 `int*`）

注意：

- 整型可用 `unsigned` 修饰
- 不同平台字节数可能不同
- 实际大小建议用 `sizeof()` 判断

### 4.2 `bool` 布尔类型

- 只有两个值：`true` 和 `false`
- 常用于关系表达式和逻辑表达式的结果
- 与整型兼容：`true` 可视为 `1`，`false` 可视为 `0`

### 4.3 枚举类型 `enum`

定义形式：

```cpp
enum weekday { sun, mon, tue, wed, thu, fri, sat };
```

特点：

- 枚举元素默认从 `0` 开始依次递增
- 可手动指定初值
- 枚举元素可按常量处理
- 整数不能直接赋给枚举变量，必要时需强制转换

典型考点：

- `day = 2;` 通常视为错误
- `day = (weekday)2;` 合法

### 4.4 结构体 `struct`

作用：

- 把不同类型的数据组合成一个整体

定义形式：

```cpp
struct Student {
    int num;
    char name[20];
};
```

成员访问：

- 对象：`.` 
- 指针：`->`

重点：

- 结构变量的存储类型、作用域、生命周期与普通变量一致
- 可用 `sizeof()` 求结构体变量占用空间

### 4.5 联合体 `union`

作用：

- 多个成员共享同一段内存

特点：

- 同一时刻通常只有一个成员有效
- 所有成员共用同一地址空间
- 常用于节省内存或进行底层数据解释

### 4.6 匿名联合

特点：

- 没有联合名
- 成员可直接访问
- 常作为结构体中的嵌套成员

### 4.7 `typedef`

作用：

- 为已有类型起别名

示例：

```cpp
typedef int natural;
typedef double area, volume;
```

## 五、字符串、字符与初始化

### 5.1 字符与字符串的区别

- `'a'` 是字符常量，表示单个字符
- `"a"` 是字符串常量，本质是包含 `'a'` 和结尾 `\0` 的字符数组

### 5.2 字符串指针

```cpp
char* p = "CHINA";
```

理解：

- `p` 存放的是字符串首地址
- 指向字符串常量首字符

### 5.3 变量初始化

常见形式：

```cpp
int a = 3;
double f = 3.56;
char c = 'a';
int i(5);
```

## 六、变量存储类型

### 6.1 `auto`

- 默认存储类型
- 自动变量离开作用域后生命周期结束

### 6.2 `register`

- 建议编译器把变量放入寄存器
- 只是“建议”，不一定真正放入

### 6.3 `extern`

- 声明变量或函数在别处定义
- 常用于多文件程序

### 6.4 `static`

- 静态存储期
- 在程序整个运行期间都存在

常考区别：

- 局部普通变量：每次进入作用域重新创建
- 局部静态变量：只初始化一次，退出作用域后值保留

## 七、表达式、类型转换与运算符

### 7.1 表达式基础

- 常量、变量、函数调用都可以构成表达式
- 表达式一定有类型，也通常有值
- 运算时要求操作数类型与运算规则匹配

非法示例：

```cpp
float a;
char* p;
// a + p;  // 非法
```

### 7.2 自动类型转换（隐式转换）

混合运算时，编译器会自动进行类型提升，通常遵循“低类型向高类型转换”：

`char -> short -> int -> long -> float -> double`

重点：

- 算术运算和关系运算中常发生隐式转换
- 赋值时右值会尝试转换为左值类型
- 逻辑判断中，非 `0` 通常转为 `true`，`0` 转为 `false`

### 7.3 强制类型转换

两种常见写法：

```cpp
float(a)
(float)a
```

作用：

- 把表达式结果转换为指定类型

典型例子：

```cpp
c = float(a) / float(b);
```

### 7.4 常用运算符分类

- 算术运算符：`+ - * / %`
- 自增自减：`++ --`
- 关系运算符：`< <= > >= == !=`
- 逻辑运算符：`! && ||`
- 条件运算符：`?:`
- 位运算符：`& | ^ ~ << >>`
- 赋值运算符：`= += -= *= /= %=`
- 成员访问：`.` `->`
- 取地址/解引用：`&` `*`
- 内存管理：`new` `delete`
- 大小计算：`sizeof`

### 7.5 `sizeof`

作用：

- 求类型、变量或表达式结果类型所占字节数

写法：

```cpp
sizeof(short)
sizeof(x)
sizeof(a + b)
```

### 7.6 自增与自减

- `++i`：先加再用
- `i++`：先用再加
- `--i`、`i--` 同理

这是高频易错点。

### 7.7 赋值表达式

特点：

- 赋值表达式本身也有值
- 赋值运算符 `=` 是右结合

示例：

```cpp
a = b = c = 5;
```

含义：

- 先执行 `c = 5`
- 再执行 `b = 5`
- 最后执行 `a = 5`

### 7.8 逗号表达式

形式：

```cpp
表达式1, 表达式2
```

特点：

- 先计算表达式1
- 再计算表达式2
- 整个表达式结果为表达式2的值

### 7.9 条件运算符

形式：

```cpp
表达式1 ? 表达式2 : 表达式3
```

执行规则：

- 若表达式1为真，结果取表达式2
- 否则结果取表达式3

示例：

```cpp
x = a > b ? a : b;
```

### 7.10 位运算

常见位运算：

- 按位与：`&`
- 按位或：`|`
- 按位异或：`^`
- 按位取反：`~`
- 左移：`<<`
- 右移：`>>`

重点理解：

- 位运算对象必须是整数类型
- 左移通常低位补 `0`
- 右移对有符号数和无符号数的高位补法不同

## 八、语句与控制结构

### 8.1 C++ 语句分类

- 声明语句
- 表达式语句
- 选择语句
- 循环语句
- 跳转语句
- 复合语句
- 标号语句

### 8.2 复合语句

- 用 `{}` 把多条语句组合在一起
- 形成一个代码块

### 8.3 三种基本控制结构

- 顺序结构
- 分支结构
- 循环结构

### 8.4 `if` 语句

常见形式：

```cpp
if (表达式) 语句;
if (表达式) 语句1; else 语句2;
```

注意：

- 判断条件中应使用 `==` 判断相等
- `if (x = 1)` 常是误写，会变成赋值，不是比较
- 推荐写成 `if (1 == x)` 可降低误写成赋值的风险

### 8.5 `switch` 语句

特点：

- 条件表达式通常为整型、字符型、枚举型
- `case` 后必须是常量表达式
- `case` 值不能重复
- 多数情况下每个 `case` 末尾应加 `break`

高频考点：

- 忘写 `break` 会继续向后执行（贯穿）

### 8.6 循环语句

#### `while`

```cpp
while (表达式) 语句;
```

特点：

- 先判断，再执行

#### `do...while`

```cpp
do {
    // 循环体
} while (表达式);
```

特点：

- 先执行一次，再判断
- 至少执行一次

#### `for`

```cpp
for (表达式1; 表达式2; 表达式3) 语句;
```

理解：

- 表达式1：初始化
- 表达式2：循环条件
- 表达式3：迭代更新

### 8.7 `break` 与 `continue`

- `break`：立即结束当前循环或 `switch`
- `continue`：结束本次循环，进入下一次判断

## 九、函数基础

### 9.1 函数的作用

- 将功能模块化
- 便于复用、维护和调试

### 9.2 函数声明与定义

函数原型示例：

```cpp
void func(int a, int b);
float foo(float a);
```

说明：

- 调用前应先定义，或先声明原型
- `return` 用于返回函数结果
- `void` 函数可无返回值

### 9.3 函数调用

形式：

```cpp
函数名(实参列表)
```

支持：

- 普通调用
- 嵌套调用
- 递归调用

注意：

- 可以嵌套调用
- 不能嵌套定义函数

### 9.4 递归

递归是函数直接或间接调用自身。

必要条件：

- 递归终止条件
- 逐步逼近终止条件

讲义中典型例子：

- 求 `x` 的 `n` 次幂

### 9.5 参数传递机制

#### 值传递

特点：

- 传递的是实参的副本
- 形参变化不会影响实参

典型现象：

- 用值传递写 `Swap(int a, int b)`，交换失败

#### 指针传递

特点：

- 传递的是地址
- 通过解引用可以修改实参

典型形式：

```cpp
void Swap(int* a, int* b);
```

#### 引用传递

特点：

- 引用是变量的别名
- 语法比指针更简洁
- 能直接修改实参

典型形式：

```cpp
void Swap(int& a, int& b);
```

### 9.6 引用 `&`

核心结论：

- 引用是已存在对象的别名
- 定义引用时必须初始化
- 一旦绑定后不能改为引用别的对象
- 可作为函数参数，也可作为返回值

示例：

```cpp
int i;
int& ri = i;
```

考试常考：

- “引用必须初始化”
- “引用不是新对象，而是别名”

### 9.7 内联函数 `inline`

作用：

- 建议编译器在调用处直接展开函数体
- 可减少参数传递和控制转移开销

注意：

- 内联函数定义通常应放在调用之前
- `inline` 是一种建议，不保证一定展开

### 9.8 默认参数

概念：

- 在函数声明时可为形参预设默认值
- 调用时若未提供对应实参，则使用默认值

示例：

```cpp
int add(int x = 5, int y = 6);
```

规则：

- 默认参数必须从右往左连续设置
- 有默认值的参数右边不能再出现无默认值参数
- 默认参数在同一作用域内只能定义一次

高频判断题：

- `int add(int x, int y = 5, int z = 6);` 正确
- `int add(int x = 1, int y = 5, int z);` 错误

### 9.9 函数重载

概念：

- 在同一作用域中，允许多个函数同名
- 通过“参数个数不同”或“参数类型不同”进行区分

注意：

- 不能只靠返回值不同来区分重载
- 也不能只靠形参名不同来区分

正确区分依据：

- 参数个数
- 参数类型
- 参数顺序（类型顺序不同）

### 9.10 操作符重载

概念：

- C++ 允许把操作符当作函数重载

常见用途：

- 自定义类型（如复数）参与 `+`、`-` 等运算

本章只需掌握：

- 知道“操作符可以重载”
- 它本质上是函数

## 十、输入输出流（I/O）

### 10.1 标准输入输出

- `cout`：标准输出（屏幕）
- `cin`：标准输入（键盘）

示例：

```cpp
int x;
cin >> x;
cout << "x=" << x;
```

### 10.2 流的概念

“流”可以理解为：

- 程序与外部设备/文件之间的数据通道

术语：

- 读：提取（从流中读取）
- 写：插入（向流中写入）

### 10.3 输出流

重要类：

- `ostream`
- `ofstream`

预定义对象：

- `cout`：标准输出
- `cerr`：标准错误输出，无缓冲
- `clog`：标准错误输出，有缓冲

### 10.4 插入运算符 `<<`

作用：

- 把数据送入输出流

示例：

```cpp
cout << "sum=" << sum << endl;
```

### 10.5 输入流

重要类：

- `istream`
- `ifstream`

### 10.6 提取运算符 `>>`

作用：

- 从输入流读取数据

示例：

```cpp
cin >> x;
```

### 10.7 输出格式控制

常见方式：

- `width()`
- `setw()`
- `fill()`

用途：

- 控制输出宽度
- 控制对齐
- 控制填充字符

### 10.8 文件流

#### 输出文件流 `ofstream`

常见操作：

- `open()`
- `put()`
- `write()`
- `seekp()`
- `tellp()`
- `close()`

#### 输入文件流 `ifstream`

常见操作：

- `open()`
- `get()`
- `getline()`
- `read()`
- `seekg()`
- `tellg()`
- `close()`

理解重点：

- `ofstream` 用于写文件
- `ifstream` 用于读文件
- `open()` 建立流对象与文件之间的联系

## 十一、本章高频易错点

### 11.1 容易混淆的基础概念

- `'a'` 是字符，`"a"` 是字符串
- `=` 是赋值，`==` 是比较
- `i++` 与 `++i` 的求值时机不同
- `true/false` 与 `1/0` 有对应关系，但逻辑含义不同

### 11.2 容易出错的语法点

- `switch` 中漏写 `break`
- 默认参数没有从右向左连续设置
- 试图只靠返回值区分函数重载
- 引用定义时忘记初始化
- 把值传递误以为能修改实参

### 11.3 容易考察的判断题/选择题点

- C++ 兼容 C，但不是纯面向对象语言
- 枚举变量不能直接接收普通整数（通常需强转）
- 位运算对象必须是整数类型
- `do...while` 循环至少执行一次
- 函数可以嵌套调用，但不能嵌套定义
- 注释不生成目标代码

## 十三、考前速记版

### 13.1 一句话抓主线

- C++ = C 的兼容扩展 + 更强的抽象能力 + 更灵活的函数机制

### 13.2 五个最重要的考点

- `=` 和 `==` 的区别
- 值传递、指针传递、引用传递的区别
- 默认参数的规则
- 函数重载的判定依据
- `switch` / 循环中的执行流程（尤其 `break`）

### 13.3 三个最重要的编程意识

- 先看“类型是否匹配”
- 再看“表达式如何求值”
- 最后看“控制流如何执行”
