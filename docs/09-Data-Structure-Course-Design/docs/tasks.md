# 项目任务清单

## 1. 使用说明

本文件用于管理小组当前任务、负责人、优先级、状态和截止时间。

状态说明：

- `todo`：未开始
- `doing`：进行中
- `done`：已完成
- `blocked`：被阻塞

优先级说明：

- `P0`：必须优先完成，直接影响项目推进
- `P1`：重要，但可在 P0 后推进
- `P2`：增强项或后置项

## 7. 阻塞处理规则

如果出现以下情况，任务状态改为 `blocked`，并在群里同步：

- 需求不清，无法继续
- 依赖文档未完成
- 字段或接口发生冲突
- 算法方案与实现方案不一致

处理原则：

- 组员 A 负责收敛问题并更新任务状态
- 组员 B 负责解释算法相关阻塞
- 组员 C 负责解释实现相关阻塞

## 8. 更新规则

- 任务状态至少每周更新一次。
- 完成任务后，负责人需同步修改本文件。
- 若新增任务，直接追加到任务表并分配负责人。
- 若调整优先级，必须在群里说明原因。
