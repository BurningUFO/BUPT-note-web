    《Ch1-introduction-2-网络分类》讲解文档

    说明：本文基于课件 Ch1-introduction-2-网络分类.pdf 的实际内容整理，并结合计算机网络基础知识做必要的讲解性补充。由于本章是“导论中的分类章节”，重点在网络形态与互联方式，不在具体协议报文头，因此本文会弱化 Header 细节，强化分类逻辑、转发机制和工程视角。

    1. 本章在课程体系中的位置

    1.1 层级定位（Layer Mapping）

    这一章严格来说不属于某一个单独协议层，而是一个“跨层的导论性主题”。

    • 如果按 TCP/IP 五层模型看，它横跨：
    • 物理层（Physical Layer）：链路介质，如双绞线、同轴电缆、光纤
    • 数据链路层（Data Link Layer）：局域网、广播式共享介质、Ethernet、WiFi
    • 网络层（Network Layer）：路由器、广域网、通信子网、网络互联
    • 如果按 OSI 七层模型看，它主要落在：
    • 第 1 层：物理层
    • 第 2 层：数据链路层
    • 第 3 层：网络层

    1.2 本章解决的核心问题

    本章回答的不是“某个协议怎么实现”，而是下面三个更基础的问题：

    1. 网络由什么组成：主机、交换设备、链路分别扮演什么角色。
    2. 网络可以怎样分类：按传输技术、按规模、按互联位置如何划分。
    3. 不同网络为什么长得不一样：不同覆盖范围、不同转发方式，决定了不同的网络技术与设备选择。

    从工程角度看，本章是在建立一个“地图”。你后面学到的 Ethernet、IP、TCP、WiFi、路由算法，都会落到这张地图里的某一块。

    2. 本章主线：网络分类的三条轴

    课件给出的主线非常清晰，计算机网络可以从三个维度分类：

    1. 按传输技术（Transmission Technology）
    2. Broadcast network vs Point-to-point network
    3. 按覆盖范围（Network Scale）
    4. PAN / LAN / MAN / WAN
    5. 按在互联网络中的位置（Position in Interconnected Networks）
    6. Access network vs Core network

    这三条轴并不是互相替代的关系，而是从不同观察角度看同一个网络系统。

    例如：

    • 家里的 WiFi 是一个 LAN（按规模看），它往往也是带有广播特征的共享链路（按传输技术看），同时它又属于接入网的一部分（按互联位置看）。

    这正是初学者最容易混淆的地方：一个网络可以同时属于多个分类维度中的某一类。

    3. 网络的硬件基础：为什么分类前先看组成

    课件在正式分类前，先介绍了网络的硬件组成，这一步很关键，因为“分类”本质上就是在区分这些组件如何被组织起来。

    3.1 核心组成

    • Hosts / End Systems（主机 / 端系统）
    • 负责产生和消费数据
    • 是通信的源和目的地
    • Switches / Routers（交换机 / 路由器）
    • 属于中间节点
    • 负责转发，而不是最终使用应用数据
    • Communication Links（通信链路）
    • 负责承载比特流
    • 可以是有线，也可以是无线

    3.2 常见链路和接口

    课件列出了典型介质与接口：

    • Twisted Pair（双绞线）
    • Coax（同轴电缆）
    • Fiber（光纤）
    • Ethernet NIC
    • Wireless NIC

    这提示了一个重要事实：

    • 网络分类不是抽象概念游戏，它直接对应物理部署和设备形态。

    比如：

    • 你使用光纤宽带接入家庭路由器，说明这条接入链路在物理层上就已经和宿舍里一根双绞线的以太网有差异。
    • 你使用手机 5G 上网，意味着这条链路天然更接近广域接入网络，而不是一个本地私有 LAN。

    3.3 有基础设施网络 vs Ad hoc 网络

    课件图里还出现了 Ad hoc network（自组织网络）。

    它对应两种不同的组织方式：

    • Infrastructure-based network（有基础设施网络）
    • 有 AP、交换机、路由器等固定设备
    • 大多数现实网络都属于这一类
    • Ad hoc network（自组织网络）
    • 节点之间直接协同
    • 不依赖固定基础设施

    这背后的核心差异是：

    • 前者把“控制与转发”交给专门设备；
    • 后者把“组网与协同”的职责部分下放给终端本身。

    4. 按传输技术分类：Point-to-Point vs Broadcast

    这是本章第一个核心分类标准。

    4.1 Point-to-Point Network（点到点网络）

    定义

    点到点链路是指：一条链路只直接连接两个节点。
    如果数据要从源主机到目标主机，通常需要经过一个或多个中间节点逐跳转发。

    课件中的关键词：

    • a packet may visit one or more intermediate machines
    • unicasting
    • WANs

    动态过程（Mechanism）

    可以把点到点网络中的一次传输理解为下面的过程：

    步骤 1：源主机把分组（packet）发送给第一跳设备。
    步骤 2：中间路由器根据目的地址做 routing decision。
    步骤 3：分组被转发到下一跳。
    步骤 4：重复上述过程，直到到达目的主机。
    步骤 5：目的主机完成解封装并交给上层应用。

    为什么这样设计

    因为在广域范围内，不可能让所有设备共享同一根“全网可见”的公共链路。
    因此，必须把大网络拆成许多点到点链路，再通过中间设备串接起来。

    字符图理解

    Host A --> R1 --> R2 --> R3 --> Host B

    这条路径上的每一跳，本质上都是一条点到点链路。

    关键工程含义

    • 可扩展性更强
    • 更适合远距离互联
    • 必须依赖路由器和路由算法
    • 主要支持 unicast（单播）

    易错点

    “点到点网络”不是说整个网络里只有两个设备，而是说：

    • 每条链路是点到点的
    • 整个网络可以由很多点到点链路拼接而成

    4.2 Broadcast Network（广播式网络）

    定义

    广播式链路是指：多个节点共享同一个传输介质。
    一个节点发出的分组，可以被该共享介质上的所有节点“看到”。

    课件中的关键描述是：

    • Sending a packet to all destinations
    • each machine checks the address field
    • If the packet is intended for the receiving machine, that machine processes the packet
    • 否则忽略

    动态过程（Mechanism）

    广播式链路中的一次发送，可以理解为：

    步骤 1：发送方把帧/分组放到共享链路上。
    步骤 2：链路上的所有节点都能接收到这份信号。
    步骤 3：每个节点检查地址字段。
    步骤 4：目标节点接收并处理；非目标节点丢弃。
    步骤 5：如果是广播地址，则所有节点都处理；如果是组播地址，则只有组成员处理。

    为什么这样设计

    在较小范围、共享介质的网络里，这种方式简单直接：

    • 不需要复杂的逐跳路径选择
    • 所有节点天然处于同一广播域或共享介质之上

    字符图理解

              +--> Host B (ignore)
Host A ---+--> Host C (receive)
          +--> Host D (ignore)

    与单播、广播、多播的关系

    • Unicast（单播）：一对一
    • Broadcast（广播）：一对所有
    • Multicast（多播 / 组播）：一对特定一组

    注意：

    • “广播式链路”描述的是底层共享传输方式
    • “广播 / 多播 / 单播”描述的是发送语义

    两者相关，但不是完全同一个维度。

    关键工程含义

    • 适合局域网（特别是传统共享介质 LAN）
    • 实现简单
    • 但共享资源意味着会有竞争、冲突和带宽分摊问题

    这也是后面你会学习 MAC、CSMA/CD、CSMA/CA 的背景。

    4.3 Point-to-Point 与 Broadcast 的核心权衡（Trade-offs）

    维度 Point-to-Point Broadcast
    链路组织方式 每条链路连接两个节点 多节点共享同一介质
    转发方式 逐跳转发 所有节点都能“看到”
    典型场景 WAN、跨区域互联 LAN、局部共享网络
    主要通信语义 Unicast 为主 可自然支持 Broadcast / Multicast
    是否依赖路由 强依赖 局部场景下依赖较弱
    可扩展性 更强 广播域过大时扩展性差
    主要代价 路由设计复杂 共享介质竞争、冲突控制复杂

    一句话理解

    • Point-to-point 解决的是“大范围怎么串起来”
    • Broadcast 解决的是“小范围怎么共享起来”

    5. 按覆盖范围分类：PAN / LAN / MAN / WAN

    这是本章第二个核心分类标准。

    课件明确强调：
    不同网络不只是覆盖范围不同，所使用的技术也不同。

    这句话很重要。
    因为很多人会把 PAN/LAN/MAN/WAN 简化成“范围从小到大”，但从工程上看，这种理解太浅。

    5.1 PAN（Personal Area Network，个域网）

    定位

    PAN 是最小范围的网络，围绕个人设备展开。

    虽然课件没有展开具体技术，但从学习框架上可以理解为：

    • 面向个人近距离设备互联
    • 范围通常是人体周边或桌面级

    典型场景

    • 手机和耳机
    • 穿戴设备和手机
    • 个人设备之间的短距离同步

    理解重点

    PAN 的重点不是“复杂路由”，而是：

    • 低功耗
    • 短距离
    • 轻量接入

    5.2 LAN（Local Area Network，局域网）

    课件定义

    课件给出的定义要点：

    • LAN is a privately owned network
    • Connecting hosts to edge router
    • Sharing ONE communication link
    • 典型技术：Ethernet, WiFi

    这几句话分别意味着什么

    1. 私有网络（privately owned）

    说明 LAN 通常由家庭、学校、公司自行部署和管理，而不是由运营商跨地区统一建设。

    2. 连接主机到边缘路由器（edge router）

    这表明 LAN 的角色常常是：

    • 先把本地终端连起来
    • 再通过边缘路由器接入更大的互联网络

    也就是说，LAN 常常是“用户内部网络”。

    3. 共享 ONE communication link

    这句带有导论化表达，它强调的是：

    • 在 LAN 中，多个主机共享某种本地接入资源
    • 这也解释了为什么局域网天然关注介质访问控制（MAC）

    典型技术

    • Ethernet
    • WiFi / WLAN

    这两个技术后面都会单独展开学习，但这里先抓住共性：

    • 都面向局部范围高效接入
    • 都强调本地互联、低成本和高吞吐

    LAN 的工程特征

    • 管理边界清晰
    • 时延较低
    • 速率通常较高
    • 节点密度较高
    • 适合文件共享、打印、办公、实验室联网

    5.3 MAN（Metropolitan Area Network，城域网）

    课件给出的信息

    • A metropolitan area network based on cable TV
    • Topology: Tree
    • Wireless: WiMax

    核心理解

    MAN 位于 LAN 和 WAN 之间，覆盖一个城市或较大城区。

    它解决的问题是：

    • 如何把一个城市范围内的多个局部网络连接起来
    • 在不进入“超大规模全国骨干网”的前提下，实现区域级互联

    为什么会出现树形拓扑（Tree）

    如果基于有线电视网（Cable TV）演进：

    • 干线从中心向外辐射
    • 下级分支不断展开

    这天然形成树状结构。

    工程视角

    • MAN 往往带有明显的运营商网络色彩
    • 规模大于一个单位内部 LAN
    • 但还没有到全国或全球 WAN 的级别

    5.4 WAN（Wide Area Network，广域网）

    课件给出的定义

    • telecommunication network that covers a broad area
    • Providing connections from a LAN to the Internet
    • 无线广域技术：3G, 4G, 5G

    核心理解

    WAN 是面向大范围互联的网络，它的本质不是“很多机器直接相连”，而是：

    • 通过中间转发设备
    • 通过大量点到点链路
    • 把分散在广域范围内的多个局部网络连接起来

    为什么 WAN 必然更复杂

    因为它要同时解决：

    • 跨地域传输
    • 多跳转发
    • 资源调度
    • 路由收敛
    • 路径选择

    这也是为什么 WAN 天然和 router、routing algorithm 强绑定。

    与无线蜂窝网络的关系

    课件把 3G/4G/5G 归为无线广域接入手段，这是合理的，因为：

    • 蜂窝网络本质上是运营商建设的大范围接入系统
    • 用户可以在较大地理范围内保持连通性

    5.5 PAN / LAN / MAN / WAN 的横向对比

    类别 覆盖范围 管理主体 典型技术 核心特征
    PAN 个人周边 个人 课件未展开 短距离、轻量、低功耗
    LAN 局部区域 家庭/学校/公司 Ethernet, WiFi 私有、高速、本地共享
    MAN 城市范围 多为运营商或区域管理者 Cable TV, WiMax 区域级互联
    WAN 跨城市/跨区域/跨国 电信运营商 3G/4G/5G，路由互联 多跳、广域、依赖路由

    真正该记住的不是“范围从小到大”

    而是：

    • 范围变大，网络组织方式会改变
    • 设备类型会改变
    • 转发机制会改变
    • 管理方式会改变
    • 成本和复杂度也会显著上升

    6. WAN 的关键补充：Communication Subnet（通信子网）

    这一页是本章很关键、但很多同学第一次学时容易跳过去的内容。

    6.1 什么是通信子网

    课件写的是：

    • Switching elements
    • Routing decisions: Routing algorithm
    • Message Path
    • Single-path vs Multi-path

    这意味着：

    • 在广域网里，真正负责把消息送出去的，不是终端主机本身
    • 而是由一组中间交换设备构成的“转发骨架”

    这个转发骨架，就是广义上理解的通信子网。

    6.2 通信子网的组成

    • Switching Elements（交换设备）
    • 典型就是 router
    • 负责分组转发
    • Links（链路）
    • 负责把交换设备串接起来
    • Routing Algorithm（路由算法）
    • 负责决定走哪条路

    6.3 动态过程：分组如何穿过通信子网

    步骤 1：源主机把分组交给接入它的第一台路由器。
    步骤 2：该路由器根据目的地址查询转发表。
    步骤 3：路由器选择下一跳链路。
    步骤 4：后续路由器重复同样过程。
    步骤 5：分组最终到达目标网络，再交给目标主机。

    6.4 Single-path vs Multi-path

    • Single-path：源到目的地通常沿一条确定路径前进
    • Multi-path：可能有多条候选路径，系统在多个路径间做选择或调度

    工程意义在于：

    • 多路径可以提升可靠性和负载均衡能力
    • 但也会增加路径管理与顺序控制的复杂度

    6.5 为什么这部分很重要

    你后面学习网络层时会反复遇到这个思想：

    • 终端只负责“我要发给谁”
    • 网络层核心设备负责“怎么把它送到”

    这正是 IP + routing 的基本世界观。

    7. Internetworks：internet 与 Internet

    这一页是概念题高频考点。

    7.1 课件原意

    • internet: network of networks
    • Internet: the unique world-wide internet

    7.2 两者的准确区别

    • internet（小写）
    • 泛指“网络的网络”
    • 只要多个网络互联起来，就可以构成一个 internet
    • Internet（大写）
    • 特指全球范围那个唯一的公共互联网，也就是因特网

    7.3 为什么这个区分重要

    因为在技术语境中：

    • “互联网技术”可以是一个通用互联架构概念
    • “因特网”则是一个具体存在的全球系统

    如果考试里问：

    • “internet 的定义是什么？”
      正确答案是“网络的网络”
    • “Internet 是什么？”
      正确答案是“全球唯一的公共互联网”

    7.4 设备层面的关键点

    不同网络之间之所以能互联，靠的是：

    • router（路由器）

    也就是说：

    • 交换机更偏向同一局域范围内的数据转发
    • 路由器更偏向不同网络之间的互联

    这也是后面 Layer 2 和 Layer 3 区别的入口。

    8. 按互联位置分类：Access Network vs Core Network

    这是本章第三个核心分类标准。

    8.1 Access Network（接入网）

    定义

    接入网是用户设备接入更大网络系统的那一段网络。

    课件给出的典型例子：

    • Home network
    • Institutional/company network
    • Mobile network

    它解决什么问题

    接入网解决的是：

    • “用户如何连上网”

    也就是从用户终端到运营商或更大互联网络之间的第一段连接。

    工程特征

    • 靠近终端
    • 节点数量大
    • 接入方式多样（有线、WiFi、移动蜂窝）
    • 更强调“最后一公里 / last mile”

    8.2 Core Network（核心网）

    定义

    核心网是承载跨区域、大规模流量转发的网络骨干。

    课件里出现了：

    • Regional ISP
    • Global ISP

    这说明核心网通常位于运营商或骨干网层面。

    它解决什么问题

    核心网解决的是：

    • “数据在大范围内如何高效穿越网络”

    也就是：

    • 长距离传输
    • 多区域互联
    • 高容量转发

    工程特征

    • 更强调吞吐能力和路由效率
    • 设备更强
    • 网络设计更复杂
    • 更关注可靠性、冗余、可扩展性

    8.3 Access 与 Core 的关系

    你可以把它们理解成：

    用户设备 -> 接入网 -> 边缘/区域网络 -> 核心网 -> 目标区域 -> 目标接入网 -> 目标设备

    接入网负责“把你接进来”，核心网负责“把你送过去”。

    8.4 Access vs Core 的横向对比

    维度 Access Network Core Network
    位置 靠近用户 位于网络骨干
    主要任务 用户接入 大规模远距离转发
    常见场景 家庭网、校园网、移动接入 区域 ISP、全球 ISP
    设计重点 接入方式、成本、覆盖 容量、可靠性、路由效率
    与用户关系 直接面向终端 通常不直接被终端感知

    9. 这章虽然不讲具体协议头，但仍有“机制”

    根据 AGENTS.md 的要求，本章即使不涉及报文格式，也要抓住“交互过程”和“机制因果”。

    本章至少有三个值得牢固掌握的机制。

    9.1 广播链路中的地址筛选机制

    步骤 1：发送方把数据放上共享介质。
    步骤 2：所有节点都能收到。
    步骤 3：每个节点检查目标地址。
    步骤 4：匹配则接收，不匹配则丢弃。

    如果没有这套地址筛选机制，会发生什么？

    • 所有节点都会把所有数据都交给上层处理
    • 造成严重的无效处理和语义混乱

    因此，地址字段的存在本质上是在共享介质中恢复“选择性接收”的能力。

    9.2 广域网中的逐跳转发机制

    步骤 1：源主机给出目的地址。
    步骤 2：第一台路由器做下一跳选择。
    步骤 3：后续路由器持续逐跳转发。
    步骤 4：数据跨越多个网络后到达目标。

    如果没有逐跳转发机制，会发生什么？

    • 终端必须直接知道全网拓扑
    • 网络规模一大就不可管理

    所以，逐跳转发本质上是把“大规模路径问题”分解为“每一跳的局部决策问题”。

    9.3 接入网与核心网的分层机制

    步骤 1：终端先通过接入网进入网络系统。
    步骤 2：流量被汇聚到更高层级网络。
    步骤 3：核心网负责远距离运输。
    步骤 4：目标侧再通过接入网下沉到终端。

    这种设计的好处是：

    • 职责分离
    • 易于扩展
    • 便于运营和分层管理

    10. 本章的“定量分析”该怎么理解

    这章课件几乎没有给出具体公式，因此这里需要明确：

    • 本章的重点不是公式计算
    • 重点是“分类标准 + 网络结构理解”

    不过，从课程衔接角度，可以先知道下面这些量会在后续章节频繁出现：

    • 传输时延：\(d_{trans} = L / R\)
    • 传播时延：\(d_{prop} = d / s\)
    • 吞吐量（throughput）
    • 带宽（bandwidth）
    • 误码率（bit error rate）

    本章最后的小结页提到 QoS（时延、带宽、误码率），它是在提醒你：

    • 网络分类不是脱离性能的
    • 不同类型网络的部署方式，最终都会影响 QoS

    例如：

    • LAN 往往更容易获得低时延和高带宽
    • WAN 因为路径长、跳数多，时延通常更高

    这属于基于网络结构得出的工程推论，不是课件显式给出的精确公式结论。

    11. 考试和面试中最容易出的点

    11.1 高频概念题

    1. 点到点链路和广播链路的区别是什么？
    2. 单播、广播、多播分别是什么？
    3. PAN / LAN / MAN / WAN 分别对应什么范围和典型技术？
    4. 为什么 WAN 更依赖路由器和路由算法？
    5. internet 和 Internet 有什么区别？
    6. 接入网和核心网分别负责什么？

    11.2 高频易混点

    易混点 1：点到点 vs 一对一

    • 点到点描述的是链路结构
    • 一对一（单播）描述的是通信语义

    二者常常相关，但不是同义词。

    易混点 2：广播链路 vs 广播报文

    • 广播链路：多个节点共享介质
    • 广播报文：一次发送给所有节点

    在广播链路上也可以发送单播报文；只是所有节点都能先“看见”，再由地址字段决定谁处理。

    易混点 3：LAN 一定是广播网络吗

    从教学入门视角，课件把 LAN 和广播式网络联系起来，这是合理的。
    但从现代工程实现看，交换式以太网已经显著降低了“所有节点共享同一冲突域”的程度。

    所以考试答题时：

    • 按课件框架回答：LAN 常与广播链路相关联
    • 但工程上要知道：现代 LAN 的实现细节可能更复杂

    易混点 4：internet 和 Internet

    这是大小写敏感概念题，最容易丢分。

    12. 用一个完整例子把三种分类串起来

    以“你在宿舍用手机连校园网，再访问外部网站”为例：

    场景拆解

    步骤 1：手机通过 WiFi 接入宿舍无线路由或校园 AP。
    这部分从规模上看，属于 LAN；从位置上看，属于 Access network。

    步骤 2：校园网边缘设备把流量送入上级运营商网络。
    这说明本地网络正在接入更大范围的互联网络。

    步骤 3：数据进入区域 ISP，再进入更高层级骨干。
    这部分从位置上看，属于 Core network；从规模上看，更接近 WAN。

    步骤 4：数据经过多个路由器逐跳到达目标服务器所在网络。
    这里体现的是 Point-to-point + routing 的广域转发机制。

    这个例子告诉你什么

    同一次访问过程里，可能同时出现：

    • LAN
    • WAN
    • Access network
    • Core network
    • 点到点逐跳转发

    所以，本章真正要建立的是“多维观察网络”的能力。

    13. 一页速记版

    13.1 三条分类轴

    • 按传输技术：Point-to-point vs Broadcast
    • 按规模：PAN / LAN / MAN / WAN
    • 按位置：Access network vs Core network

    13.2 三个关键词

    • LAN：私有、本地、常见技术是 Ethernet 和 WiFi
    • WAN：广域、多跳、依赖 router 与 routing
    • Internet：全球唯一的因特网

    13.3 三个高频易错点

    • 点到点是链路结构，不是“只有两台设备”
    • 广播链路不等于所有主机都处理数据
    • internet 和 Internet 不是一个概念

    14. 本章最终总结

    这章表面上在讲“分类”，本质上在讲的是：

    • 网络是如何被组织起来的
    • 不同组织方式为什么需要不同技术
    • 从局部共享到广域互联，网络复杂度是如何一步步上升的

    如果你把这章真正学透，后面所有章节会更容易建立坐标感：

    • 学 Ethernet 和 WiFi 时，你知道它们主要在 LAN / Access 场景下工作
    • 学 IP 和路由时，你知道它们是 WAN / internetworking 的核心能力
    • 学 MAC 时，你知道它是在解决共享介质下的接入控制问题

    因此，这章不是“背定义”的章节，而是整个《计算机网络》课程的地图入口。
